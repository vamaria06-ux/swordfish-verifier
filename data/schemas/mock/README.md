# Пустая директория схем для mock-сервера

Эта директория намеренно не содержит файлов JSON-схем SNIA.

`config_mock.yml` указывает `schemas_dir` сюда, поэтому автоматический
парсер (`verifier/parser.py`) не находит файлов схем для шести базовых
ресурсов и верификатор откатывается на встроенные (built-in) правила
для ServiceRoot, Systems, StorageSystems, StoragePools, Volumes и Drives.

Это сделано намеренно: встроенные правила описывают ресурсы как
коллекции (с полями `Members`/`Members@odata.count`), что соответствует
эндпоинтам, которые реализует `mock_server/broken_server.py`. Реальные
JSON-схемы SNIA (в `data/schemas/json-schema`) для Volume/StoragePool
описывают отдельный элемент коллекции, а не саму коллекцию, поэтому их
использование против mock-сервера приводило бы к ложным несоответствиям.

Для запуска против настоящего эмулятора Swordfish используйте
`config.yml`, где `schemas_dir` указывает на полный каталог схем SNIA.
